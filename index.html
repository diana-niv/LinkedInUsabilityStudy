Abstract

This study evaluated LinkedIn's mobile and desktop usability for university students and recent graduates looking for jobs. We used a Heuristic Evaluation followed by a counter-balanced user study with ten participants. Key issues included poor Consistency and Standards, a bad Match between System and the Real World, and a lack of User Control and Freedom — all significantly worse on mobile.

01

Introduction

LinkedIn's main goal is to help people build their professional profiles, grow their networks, and connect with opportunities that move their careers forward. For our study, we focused on students and asked them to complete tasks related to the system's usability: finding a job, returning to a saved job, and exploring events by finding, attending, and leaving them.

We then compared how these tasks performed on the desktop site versus the mobile app, targeting university students and recent graduates as our primary user group.

02

Heuristic Evaluation

We evaluated the LinkedIn system through Nielsen's 10 usability heuristics. The following problems were identified as very important for our target group:

My Jobs page When a user saves a job, they can find it on "My Jobs". On both Mobile and Desktop there is a mismatch between the names of "Saved" and "My Jobs". 4. Consistency and Standards Minor

Study Design

The study employed a within-subject, counter-balanced design across Desktop and Mobile conditions. Every participant completed the same tasks on both platforms. Half started on Mobile, then moved to Desktop; the other half did the reverse. Sessions were screen-recorded; SEQ and SUS questionnaires were administered via Google Forms.

A clean LinkedIn profile was created for each session to ensure task performance was not influenced by existing user data.

Tasks

Three core tasks were chosen to represent the key user goals of our target group:

  • Task 1 – Find a Job and Apply: standard function test.
  • Task 2 – Save a Job and Open Saved Jobs: participants had to avoid using the pop-up shortcut to test native menu navigation.
  • Task 3 – Find, Attend, and Leave an Event: search function was prohibited to test organic discoverability.

SEQ Score Summary

The SEQ (Single Ease Question) results reinforced the SUS findings. Desktop performance was consistently rated higher across all three tasks. The most dramatic gap appeared at Task 3, where mobile participants reported far greater difficulty — many scoring it at the lowest possible ease rating.

05

Conclusion

User testing fully supported our claims from the heuristic evaluation. The same usability issues appeared in real user behavior, confirming the validity of our initial findings. What became obvious very quickly is that desktop generally supports tasks well, while mobile users struggled significantly — often becoming confused or frustrated.

1
Saved Jobs Are Harder to Find Than They Should Be

LinkedIn uses two different labels for the same feature ("Saved" vs. "My Jobs"). This caused real confusion — participants clicked around different menus, hesitated, or opened wrong pages before finding their saved jobs. Even more pronounced on mobile.

2
The Events Feature Is Hidden — and Users Really Feel It

LinkedIn makes it surprisingly difficult to find Events, especially on mobile. Participants had to dig through "Manage my network" or scroll the homepage at length, with several noting they had never used Events before. This single issue became the biggest barrier in Task 3.

3
Leaving an Event Isn't Straightforward

The "Leave event" option is tucked behind a three-dot menu. Almost no one found it quickly. Several participants weren't sure if they had done it correctly, and mobile users struggled the most — some failing the task entirely.

Limitations & Challenges

Sample Homogeneity All participants were students from the same university, limiting the range of user perspectives and generalisability.
Time Constraints A tight deadline prevented running additional test sessions, which would have increased statistical robustness.
